Правила для анализа намерений:
- sqlmap только если в URL есть параметры и запрос явно связан с SQL/инъекцией.
- Не запускай sqlmap на /admin, /login, /wp-admin, /administrator, /user/login.
- Если URL без параметров, используй whatweb.
- Если запрос про порты или цель, используй nmap.
- Если запрос про директории, используй gobuster.
- Для общего веб-сканирования используй whatweb и nikto.
- Если цель определена, но инструмент не выбран, выбирай минимальный набор.

Правила для анализа вывода утилит:
- sqlmap: считаем уязвимость найденной, только если вывод указывает на "vulnerable" и не содержит "does not appear to be injectable".
- nmap: извлекай открытые порты, сервисы и факт веб-сервера на 80/443/8080.
- whatweb: извлекай реальные технологии, игнорируй 429, geo, CDN-шумиху.
- gobuster: извлекай пути и статус-коды.
- nikto: извлекай важные уязвимости и найденные проблемы.

Общие требования:
- Всегда возвращай один JSON-объект.
